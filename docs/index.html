What is Gaia?

Think of it as a Pokédex for AI agent abilities — but open, evidence-backed, and community-owned.

○
Basic Skills
The genome of every agent. Primitive, indivisible capabilities — Tokenize, Retrieve, Classify, Embed Text.
◇
Extra Skills
Emerge when 2+ basics combine — RAG Pipeline, Autonomous Debug, Research. More than the sum of their parts.
◉
Unique Skills
Graph-isolated singularities that reached elite status through depth alone — Fine-Tune. Mastered at 4★+ with no fusion path.
◆
Ultimate Skills
High-complexity emergent capabilities found in <1% of agents — Autonomous Research Agent, Full-Stack Developer.

The Rank System

Skills level up through evidence, not declaration.

Level Rank Evidence What it means
0★ Basic None Universal LLM primitive — any capable model does this by default
1★ Awakened None Catalogued and available — the foundation tier
2★ Named Class C First confirmed demonstration by a real contributor
3★ Evolved Class B Reproducible, fully documented
4★ Hardened Class B or A Failure modes known; battle-tested
5★ Transcendent Class B or A Composable and self-improving
6★ Transcendent ★ Class A Apex — peer-reviewed, named to the agent who unlocked it

Get Started

Three steps from zero to your first skill tree.

1

Install the CLI

Puts the gaia command on your PATH.

pip install gaia-cli
pip install gaia-cli
🪟If gaia isn't recognized after install, run the command below in PowerShell, then restart your terminal.
$env:PATH += ";" + (python -c "import sysconfig; print(sysconfig.get_path('scripts', 'nt_user'))")
🚀To instantly pull the latest registry and update the core CLI anytime, run gaia update.
2

Initialize & scan

Inside any project repo, run gaia init to set up tracking, then gaia scan to detect which skills your agent demonstrates.

gaia init
gaia scan
💡Your GitHub username and scan paths are auto-detected. Pass --user your-username to override.
3

Submit your skills

Push your detected skills to the registry for maintainer review. A GitHub PR is opened automatically.

gaia push
💡Run gaia push --dry-run first to preview what would be submitted.

Connect Gaia to Your Agent

Install the MCP server with copy-paste commands; keep agent-specific JSON in the MCP package docs.

M

Claude Code

Add Gaia as an MCP server from the CLI.

claude mcp add gaia -- npx @gaia-registry/mcp-server
*

Any MCP-compatible agent

Use command npx with argument @gaia-registry/mcp-server. Set GAIA_USER to your GitHub username if your client supports environment variables. 

npx @gaia-registry/mcp-server
💡Need Claude Desktop, Cursor, VS Code, Codex CLI, Gemini, or OpenClaw config-file examples? Open packages/mcp.

After scanning, appraise your detected skills to see which ones are eligible for promotion, then promote them.

gaia appraise your-skill
gaia promote your-skill

Quick Glossary

Terms you'll see throughout the registry.

gaia.json
The canonical source of truth. Every skill and its connections live here.
named skill
A real contributor implementation of a canonical skill (e.g. karpathy/autoresearch).
prerequisites
Skills that must be present before an extra or ultimate skill can emerge.
fusion
When a set of prerequisites unlocks a higher-tier skill.
evidence class
A–C quality tiers for skill demonstrations. A = peer-reviewed, B = reproducible demo, C = credible demo.
skill tree
Your personal record of skills — tied to your GitHub username and portable across repos.
[Unclaimed ✦]
An Ultimate skill no one has built yet — open for the taking.
