What is Gaia?

Think of it as a Pokédex for AI agent abilities — but open, evidence-backed, and community-owned.

○
Basic
Basic Skills
The genome of every agent. Primitive, indivisible capabilities — Tokenize, Retrieve, Classify, Embed Text.
◇
Extra Skill
Extra Skills
Emerge when 2+ basics combine — RAG Pipeline, Autonomous Debug, Research. More than the sum of their parts.
◆
Ultimate Skill
Ultimate Skills
High-complexity emergent capabilities found in <1% of agents — Autonomous Research Agent, Full-Stack Developer.

The Rank System

Skills level up through evidence, not declaration.

LevelClassRankEvidenceWhat it means
0 FUnawakenedNone Universal LLM primitive — any capable model does this by default
I DAwakenedNone Catalogued and available — the foundation tier
II CNamedEvidence Tier C First confirmed demonstration by a real contributor
III BEvolvedEvidence Tier B Reproducible, fully documented
IV AHardenedEvidence Tier B or A Failure modes known; battle-tested
V STranscendentEvidence Tier A Composable and self-improving
VI ★ SSTranscendent ★Evidence Tier A Apex — peer-reviewed, named to the agent who unlocked it

Get Started

Five steps from zero to your first skill scan.

1

Clone the registry

Grab your local copy of the canonical skill graph.

git clone https://github.com/mbtiongson1/gaia-skill-tree.git
cd gaia-skill-tree
2

Install the CLI

One pip install puts the gaia command on your PATH.

# Core install
pip install -e .

# Strongly recommended — enables semantic search
pip install -e ".[embeddings]"
gaia scan          # ~30 sec, run once
# Windows only — if `gaia` is not recognized after install, run in PowerShell:
$env:PATH += ";" + (python -c "import sysconfig; print(sysconfig.get_path('scripts', 'nt_user'))")
# To persist: add that path via System → Environment Variables → Path
3

Initialize your project

Inside the repo you want Gaia to track, run gaia init. It creates a .gaia/config.json file, auto-detecting your GitHub username and common skill paths.

gaia init
# Username auto-detected from git remote / git config
# Scan paths auto-detected from AGENTS.md, .claude/skills/, etc.
gaia init --user your-username   # override if needed
{
  "gaiaUser": "mbtiongson1",       // auto-filled
  "scanPaths": [".claude/skills"]  // auto-detected
}
4

Scan your repo

Gaia scans your configured paths and reports which canonical skills your agent demonstrates, plus any fusion opportunities.

gaia scan
# Prints detected skills and combination candidates
5

Submit a batch for review

Preview what would be submitted with --dry-run, then push to create an intake record.

gaia push --dry-run   # preview the JSON
gaia push             # write intake + open a PR
💡 Tip — semantic search

Once embeddings are generated, gaia skills search "autonomous research" finds skills by meaning, not just keywords. The model runs fully locally — no API key needed.

Install a named skill

Other contributors publish real implementations of generic skills. Install one directly into your repo:

gaia skills install karpathy/autoresearch
gaia skills list          # see what's installed
gaia pull                    # pull latest versions
★

Promote your own named skill

Once your pushed skill is awakened, claim it under your GitHub username:

# Maintainers review registry-for-review batches before updating registry/gaia.json
gaia push             # submit for maintainer review
✦

Claim an unclaimed Ultimate skill

Five Ultimate skills show ◆ /skill-id [Unclaimed ✦] in the registry — no contributor has implemented them yet. Build one, push it through intake review, then name it to claim the title slot and become the origin contributor:

# Find unclaimed ultimates — look for [Unclaimed ✦] in registry.md
# Build your implementation, then:
gaia push                    # submit intake record
# After maintainer marks it awakened:
gaia push             # submit for maintainer review
# The seed title becomes visible once your PR merges
◆+

Propose a new Ultimate skill

The highest tier. Requires ≥3 prerequisite skills, ≥3 Evidence Tier A/B sources, and 2 Core Maintainer approvals. Use the new_ultimate_skill PR template and validate locally first:

# In registry/gaia.json — add your skill with:
#   "type": "ultimate"
#   "prerequisites": ["skill-a", "skill-b", "skill-c"]
#   "evidence": [... 3× Evidence Tier A or B sources ...]
#   "status": "provisional"  (maintainers set to validated at merge)
python scripts/validate.py   # must pass before opening PR

Quick Glossary

Terms you'll see throughout the registry.

gaia.json
The canonical source of truth. Every skill lives here.
registry-for-review/
Staging area for proposed skills before they enter the graph.
registry/named/
Real implementations, attributed to contributors (e.g. karpathy/autoresearch).
prerequisites
Skills that must be present before an extra or ultimate skill can emerge.
fusion
When a set of prerequisites unlocks a higher-tier extra or ultimate skill.
evidence tier
A–C quality tiers for skill demonstrations. Evidence Tier A = peer-reviewed.
origin: true
Marks the first contributor to publish a named implementation of a skill.
gaia scan
Runs a scan and refreshes local promotion and tree outputs.
[Unclaimed ✦]
An Ultimate skill with a hidden seed title but no contributor implementation yet — open for the taking.
