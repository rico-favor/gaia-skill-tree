Frontend Cohesion Overhaul — Sampler Index

The design - system contract

Eight stages, one source of truth. Every token, rank chip, plaque variant, explorer view mode, HUD canvas, markdown tree glyph, and mobile breakpoint in this overhaul reads from registry/gaia.json and docs/css/tokens.css — generated automatically from the canonical skill graph so the visual layer can never drift from the data. Each sampler below is a live, navigable snapshot of one stage's deliverables: open it, resize it, and inspect the elements to verify the contract is unbroken.
